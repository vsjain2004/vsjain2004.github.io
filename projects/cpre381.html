Processor Design for CprE 381 - Fall 2023

CprE 381 is a course on Computer Organization and Assembly level Programming focused on the basics of processor design and MIPS architecture. There were two major projects in the course during which three different processor designs were developed to fit a subset of the MIPS architecture. The projects are usually done in a team of 3 people, but I did all of these projects by myself. All designs were programmed in VHDL and were tested in Modelsim. The designs then went through a timing analysis in Quartus Prime based on a selected FPGA hardware. All of the processors were designed according to the Harvard Architecture, as it is the most common design for the lowest level of a processor.

The first project encompassed the creation of a Single-Cycle Processor. I started out with a rough schematic for the required instructions, and wrote down all control signal accordingly. Then, the components were created, one at a time. These were assembled together alongside a file acting as the main memory of the processor. This gave birth to the final schematic, which can be seen in the figure on the right. The code for the project is also available in my GitHub here.

Using the Single-Cycle Processor as the base, I started on the second project. The first part was to create a 5-stage pipelined processor that was software scheduled. This meant that the processor would not be checking for any collisions between two instructions and that it would depend on the provided assembly program to have enough no-ops (pauses in execution) to bypass hazards. One of the biggest changes done was adding registers to save information from one stage to another. Instead of making 4 separate registers, I combined them into one component, which proved to be equivalent upon analysis. The code for ths processor can be found here.

For the second part of Project 2, I used the processor from part 1 of this project and added hazard checking functionality. This meant that more data will have to be saved, and a new component would have to be created to check for any clashes. This was done successfully with the help of process statements. The code for this is available here.

Through the projects, I was able to gain experience with designing processors. I was also able to learn a lot about VHDL and timing analysis. This experience will help me a lot in the future, as I plan to work in processor design, catered towards machine learning.
